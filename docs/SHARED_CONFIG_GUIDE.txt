# Shared Configuration System

ACCV-Lab uses a shared configuration file to keep the build system and documentation in perfect sync.

## The Configuration File: `namespace_packages_config.py`

Located in the project root, this file contains the master list of namespace packages:

```python
NAMESPACE_PACKAGES = [
    'accvlab.examples',
    # Add new namespace packages here
]
```

## What Uses This Configuration

1. **`setup.py`** - For building and installing namespace packages
2. **`docs/conf.py`** - For Sphinx documentation configuration  
3. **`docs/generate_new_namespace_package_docs.py`** - For generating documentation

## Benefits

- ✅ **Single Source of Truth**: One file controls everything
- ✅ **No Duplication**: Never get out of sync between build and docs
- ✅ **Simple Maintenance**: Add new packages in one place
- ✅ **Consistent Behavior**: Build and docs always match

## Adding a New Namespace Package

1. **Add to the list** in `namespace_packages_config.py`:
   ```python
   NAMESPACE_PACKAGES = [
       'accvlab.examples',
       'accvlab.algorithms',  # Add this line
   ]
   ```

2. **Everything else is automatic**:
   - Build system will include it
   - Documentation will be generated for it
   - No need to update multiple files

## Usage Examples

```bash
# Check current configuration
python3 namespace_packages_config.py

# Build with current configuration
pip install -e .

# Generate docs with current configuration  
cd docs && make html
```

This system ensures that what you build is exactly what gets documented, with no possibility of the two getting out of sync.
